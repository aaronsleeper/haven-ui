## What this is

On Mon 2026-05-11, Cena Color System v2 landed:
- Source spec: `Lab/cena-health-brand/_tokens/color-system-v2.md` + `Lab/cena-health-brand/_tokens/generated/palette.css` (commits `8614628` → `5d1b5e5` in cena-health-brand).
- Integration into haven-ui: commit `45e8e14` "feat(haven-pl): integrate Cena Color System v2 palette — 18 OKLCH-derived families" in haven-ui.

The integration replaced `packages/design-system/src/styles/tokens/palette.css` with the v2 generator output. v2 produces only the standard Tailwind stops (50/100/200/300/400/500/600/700/800/900/950) per family. The family is **near-neutral, low-chroma, slightly warm** (OKLCH linear-in-L interpolation between root/min/max).

**The integration commit preserved 4 pre-v2 sand stops manually for compatibility:**

> *"Manual additions preserved from pre-v2 palette: Extended sand stops (15/16/150/250) for semantic.css compatibility."*

These four stops carry a **different hue character** than v2 — they're warmer-tan, generated by a different system before v2 landed:

| Token | Hex | Family fit |
|---|---|---|
| sand-15 | `#edeceb` | very-light cool — off-family |
| sand-16 | `#f2f2f2` | very-light neutral — off-family |
| sand-150 | `#e7ded2` | **warm tan — off-family** (most visible drift) |
| sand-250 | `#d8cec0` | warm tan mid — off-family |

They sit alongside v2-canonical stops, which produces visible hue mismatches when two surfaces from different generations sit adjacent — e.g., a sand-100 nav (v2, near-neutral) next to a sand-150 ground (pre-v2, warm-tan) reads as two separate palettes.

Aaron flagged this during the agentic-shell brand-fidelity refresh on 2026-05-12 — the surface-luminance treatment (option A, darker ground + no border) had been implemented with the pre-v2 sand-150 stop, which broke the v2 family. The agentic-shell example has been corrected to sand-200 (v2-canonical). **The rest of the pattern library remediation is this handoff.**

## The decision needed

Per-stop, the question is **regenerate via v2 OKLCH or retire and sweep**:

**Regenerate**: pass each pre-v2 stop's intended L value through v2's OKLCH linear-in-L interpolator (the generator at `cena-health-brand/_tokens/generated/palette.css` + the source spec at `_tokens/color-system-v2.md`). Output:
- sand-15 ~ L 91–92% — would land near sand-100 in v2 family (near-neutral)
- sand-16 ~ L 91–92% — same as above
- sand-150 ~ L 87% — would land near `#dcdac6`-ish (near-neutral, slightly cool relative to current warm tan)
- sand-250 ~ L 78% — would land near `#c6c1b8`-ish

**Cost of regenerate:** consumers that visually depended on the warm-tan character of sand-150 / sand-250 will look different. Whether that's a regression or an improvement depends on brand intent.

**Retire and sweep**: delete the pre-v2 stops from palette.css, sweep each usage to the nearest v2-canonical stop, update semantic aliases:
- sand-15 → sand-100 (slight darkening for empty progress states)
- sand-16 → sand-100 (same)
- sand-150 → sand-200 (matches the agentic-shell ground fix done this session)
- sand-250 → sand-300 or sand-200 (depending on contrast needs per usage)
- Update `--color-surface-raised` to map to sand-200 instead of sand-150
- Update `--color-border-default` to map to sand-300 (or sand-200) instead of sand-250

**Cost of retire-and-sweep:** ~15–25 usages need per-usage decisions. Visual diff against the warm-tan baselines may show drift; conform:visual baselines will need rebaselining.

**Third option — add a warm-surface family to v2**: if the warm-tan character of sand-150 was intentional and load-bearing, the right answer might be to extend the v2 generator to add a "warm-surface" or "stone" family (already named in DESIGN.md as a tag-semantic family — could be promoted to a surface family). This is a cena-brand v2 spec decision, not a haven-ui PL patch.

## Resolution — 2026-05-12 (same-day)

Brand-fidelity verdict: **accidental drift, not load-bearing** (Option 1 — retire and sweep). Evidence: DESIGN.md names sand-150/sand-250 by function only, not chromatic principle; v1 cena-brand `color.md` describes warm neutrals as "warmth is felt, not seen" at C:0.003–0.018 (v2's sand max chroma 0.0160 is inside range); pre-v2 stops carried 3-4× the R-B spread of v2 neighbors (structural family-fit failure); no canonical doc anywhere names warm-tan as load-bearing.

Sweep applied per the remap recommended in §"The decision needed":
- `sand-15` → `sand-100`
- `sand-16` → `sand-100`
- `sand-150` → `sand-200`
- `sand-250` → `sand-300`
