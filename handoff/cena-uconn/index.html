Patient app · clickable preview

Cena × UConn — patient app

Self-contained HTML/CSS/JS, served as a static site (no backend). Click into any surface and walk its flow. Resize the window to see the responsive shell: bottom-nav on mobile (<1024px), sidebar on desktop. Built through the haven-ui pipeline against IA-v1 (Home · Order · Activity).

Tip: each flow's primary button advances to the next step; the bottom tabs switch surfaces.

Home · the front door — greeting + agent-surfaced "things to do"

The Surfacer's home. Three states. Interactive: on the one-item / several states, "Not now" dismisses the card (remembered) and the zone falls back to the calm caught-up state (home.js).

  • Caught up — the designed-for default (nothing due)
  • One item due — a single focus card
  • Several due — one focus card + "a couple of other things"

Onboarding / consent · first-run gate (cap-12) — nav suppressed

The one-time consent gate before Home unlocks. Walk it in order.

  • 1 · Welcome → 2 · Consent (HIPAA) → 3 · Preferences → Settled → Home

Order · the meal-ordering surface + fulfillment status

Order status & optimize

Post-submit pickup confirmation, live fulfillment timeline, an in-budget swap, and issue escalation. IA-v1 3-tab shell.

  • Confirmed — order placed, awaiting kitchen
  • Active — being prepared (pickup timeline)
  • Optimize — one swap to consider (comparison)
  • Issue — something's wrong → talk to a person

Meals — menu, cart, at-a-glance

Weekly ordering: entry → state-read → preferences → cart → submit → at-a-glance. Note: still on the demo agentic shell + 5-tab nav — flagged for the same re-wrap recall just got.

  • Returning patient (full flow) · First-time patient
  • Step 1 entry · 3 prefs · 4 cart · 6 submit · 7 at-a-glance

Check-ins · agent-pushed clinical moments (no tab)

Dietary recall (cap-8)

24-hour multiple-pass interview. Rebuilt on the canonical patient shell (app-shell, nav suppressed): conversation + food-list working surface. Two-region on desktop, single-column on mobile.

  • Entry → Pass 1 quick list → Pass 3 time → Pass 4 portion → Review → Confirm

Satisfaction survey (cap-9)

M3/6/9 questionnaire (reuses the assessment runner). No score shown to the patient. Items are placeholders pending Vanessa.

  • Entry → Preflight → Questions → Confirm

Assessments (cap-7)

HFIAS · WHOQOL-HIV BREF · GNKQ-R · pre-enrollment screener. Runner template + resolved single-page instances.

  • Runner: entry → preflight → question → confirm
  • Resolved: screener · HFIAS · WHOQOL · GNKQ-R

My Health & Appointments · promoted-from-Home rows, not tabs

My Health — log a check-in (cap-20)

Self-reported measures (weight, BP, A1C) + note. Interactive form: type, toggle units, Save.

  • Fillable → Saved · narrative demo

Appointments — request (cap-21)

Request-only v1 (no live calendar). Empty → request → pending → upcoming.

  • Empty → Request → Pending · Upcoming
