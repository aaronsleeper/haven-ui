Colors

Haven maps palette colors to semantic roles. Always use the semantic name in application code; raw family names are for documentation and edge cases only.

Color Role Guide

Primary (Teal)Brand actions, links, active states, focus rings
Sage (Warm Green)Patient-facing, nutrition, food, community — the brand's hue-shift family (Cena-canonical 18th family)
Secondary (Sand)Neutral surfaces, secondary buttons, subtle borders
Success (Green)Active/enrolled status, positive outcomes, confirmed states
Warning (Amber)Pending actions, approaching thresholds, caution states
Error (Red)Critical alerts, flagged metrics, destructive actions, validation errors
Info (Cyan)Informational notices, delivery-related contexts, neutral highlights
Accent (Violet)AI-generated content, insight badges, special callouts, data / inference
OrangeTerracotta / earth accent (v2 — no semantic alias yet)
YellowIllumination accent (v2 — no semantic alias yet)
LimeSage-adjacent accent (v2 — no semantic alias yet)
EmeraldAlternate saturated green (v2 — no semantic alias yet)
BlueDeep slate (sky-replacement; v2 — no semantic alias yet)
IndigoExternal systems, partner identities (v2 — no semantic alias yet)
PurpleAlternate violet (v2 — no semantic alias yet)
FuchsiaVivid magenta accent (v2 — no semantic alias yet)
PinkWarm-shifted pink (v2 — no semantic alias yet)
RoseCommunity / relational (v2 — no semantic alias yet)

v2 families without a semantic alias are available as raw utility classes (bg-orange-500, etc.) and are reserved for data visualization, illustrations, and forthcoming role assignments. Don't invent semantic vocabulary for them ad-hoc — promote through brand review.

Logo Identity Tokens

Hex-frozen per spec §5; never computed. Family palette stops decouple from these by design (spec §1.5) — use --logo-* when exact logo hex is required (SVG export, email signatures, brand assets); use the family palette stops for everything else.
