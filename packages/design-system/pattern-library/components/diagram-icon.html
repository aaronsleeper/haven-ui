Diagram Icon

Semantic icon resolver. SVG context uses text-content authoring with font-family token; HTML context uses ::before pseudo-element resolver classes for true name-only authoring.

SVG path — text-content authoring (font-family from --font-icon-family token)

In SVG, the diagram-icon class applies font-family + font-feature-settings; the AUTHOR places the icon name as text content. Material Symbols ligatures render the glyph. When FA Pro lands, a per-diagram audit replaces names with codepoints.

HTML path — name-only authoring via diagram-icon--{name} resolver class

In HTML (e.g., diagram-long-desc, Phase 2 stack-from-data mobile rendering), the resolver class sets ::before content. When the icon stack swaps from Material to FA Pro, the resolver mappings update in one file — no per-instance audit.
