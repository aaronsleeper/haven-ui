Diagrams

Brand-aligned inline-SVG primitives for system diagrams. Compose with the dagre-backed Layer 2 helper for connectivity-heavy graphs, or hand-coordinate for simple cases. See COMPONENT-INDEX.md for the canonical class list.

Choosing a primitive

  • Wrapper — start every diagram with a diagram-frame.
  • A surface (system, role, or labeled rectangle) → diagram-box + variants.
  • An inline chip inside a single box ("PHASE 1", "contracts") → diagram-pill.
  • A region overline grouping multiple boxes ("SUBSTRATE", "SURFACES") → diagram-lane.
  • A connector between elements → diagram-arrow referencing markers from diagram-marker-defs.
  • An explanatory line below the diagram → diagram-caption (Lora italic).
  • A timeline circle (done / progress / queued) → diagram-milestone.
  • An icon — SVG <text class="diagram-icon">name</text> or HTML <span class="diagram-icon diagram-icon--name">.

Layer 1 vs Layer 2

Hand-coordinate (Layer 1) when the diagram has ≤3 element types, ≤10 connectors, and no curved connectors with markers. At or past those thresholds — or whenever the layout is an explicit DAG — switch to diagram-graph (Layer 2, dagre-backed). Genuinely illustrative one-offs that don't compose from primitives land in the Phase 2 diagram-asset escape hatch (Layer 3).
