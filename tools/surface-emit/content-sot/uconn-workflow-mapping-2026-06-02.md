## The SoP set — row by row

Ten rows. For each, four things: what the SoP is for · where the steps live · open decisions · what we honestly don't know.

:::card{title="1. Care Coordinator — Marrero owns"}

**Goal.** Run the weekly cadence for the participant roster — check-ins, contact logging, escalation routing, and the PHQ9 + 3-item anxiety screenings the CC owns by IRB protocol.

**Where each step lives.**
- *Cena* — care coordinator app for roster, weekly check-in capture, contact logging, PHQ9 + anxiety admin forms.
- *Athena* — appointment scheduling (CCs book in Athena when a participant requests a visit).
- *Paper* — none in the steady-state weekly routine; falls back here only when the digital path is unavailable.

**Open decisions.**
- *Daryl-vs-Ryan workflow* — which system holds the CC's day-to-day data (see hot decision below).
- *First action on a safety-concern call* — the CC SoP's escalation block currently says "per the UConn escalation protocol" without naming the first action. Marrero's call.
- *PHQ9 positive-response thresholds* — what triggers same-day escalation vs next-week follow-up.

**What we honestly don't know.**
- Whether the CITI training the CCs hold covers PHQ9 administration scope under Connecticut licensure, or whether a credentialed mental-health provider must administer / co-sign. Pending Healthcare Data Governance review.
:::

:::card{title="2. Registered Dietitian — Marrero owns"}

**Goal.** Run the counseling sessions, build the care plan and meal plan, administer the baseline assessments (HFIAS, HEI, WHOQOL-HIV BREF, NKQ) at the in-person first visit, capture encounter notes.

**Where each step lives.**
- *Cena* — RD app, care plan, meal plan, baseline assessment forms.
- *Athena* — encounter notes; signed-consent PDFs (after scan).
- *Paper* — written consent signing at the in-person first visit (then scanned to PDF).

**Open decisions.**
- *Meal plan vs care plan* — are these two distinct artifacts in Cena's workflow, or one combined artifact?
- *Modality default* — first visit is in-person; sessions 2-5 default to virtual per M3, but tech-limited fallback path needs operational rules.
- *Dr. Woo's role* — full name verify, scope on dietary requirements.

**What we honestly don't know.**
- How dietary-requirements review fits between Dr. Woo's input and the RD's meal plan.
:::

:::card{title="3. Administrator — Vanessa owns"}

**Goal.** Keep training current, provision role-based access, distribute materials, fulfill data-export requests, digitize signed consents, manage subcontractor onboarding.

**Where each step lives.**

- *Athena* — signed-consent PDF storage.
- *Paper* — signed-consent forms before scan; subcontractor BAA paperwork.

**Open decisions.**
