The UConn pilot's program-specific compliance. The **org-level regulatory floor** (HIPAA, Common Rule citations) lives one level up under [Governance & compliance](./governance.html) — this page covers what's specific to *this* study and references that floor rather than restating it.

## The IRB gate (the load-bearing one)

Enrollment cannot start before IRB approval — a Common Rule hard gate (45 CFR 46.109, 46.116). This is research, not QI (named PI, protocol-for-IRB, generalizable-knowledge design), so plan for full-board review. Build and dry-run everything that doesn't touch a live patient meanwhile.

- Full gating sequence, review-track ranges, and named unknowns: [Launch timeline & gates](./uconn-timeline.html).
- The "30 ready in June" signal is *identified/willing*, not *consented/enrolled* — the latter is what the IRB gate governs.

## PHI / BAA / consent

- **BAA / IRB reliance** — prep now, execute in parallel with IRB review (ledger: *BAA / IRB reliance agreement*). The long pole only if left late.
- **Data tool + PHI path** — the requirement is *portability* (Cena must export/ingest for continuity of care); default to UConn Qualtrics unless a hard reason (ledger: *Data-collection tool + UConn IT/PHI*).
- **Consent capture** — CC + Athena own (cap-12).
