Motion & Animation

Haven defines duration and easing tokens for consistent, purposeful motion across all components. Every transition should use these tokens rather than arbitrary values.

Motion Principles

PurposefulMotion communicates state changes. Never animate for decoration alone.
QuickMost interactions use --duration-fast or --duration-normal. Users should never wait for an animation to finish.
ConsistentSame type of change = same duration + easing. Hover states are always fast. Overlays are slow.
AccessibleRespect prefers-reduced-motion. All transitions collapse to instant when the user prefers reduced motion.

When to use each duration

--duration-instantState toggles with no visual transition (e.g. checkbox check)
--duration-fastHover color changes, focus rings, icon swaps
--duration-normalButton press feedback, card hover lift, tooltip appear
--duration-slowSidebar slide, dropdown open, modal enter, accordion expand
--duration-slowerPage transitions, large layout shifts, skeleton-to-content reveal

Easing Curves

All demos use --duration-slow (300ms) so the easing difference is visible.

--ease-default

cubic-bezier(0.4, 0, 0.2, 1)

--ease-in

cubic-bezier(0.4, 0, 1, 1)

--ease-out

cubic-bezier(0, 0, 0.2, 1)

--ease-in-out

cubic-bezier(0.4, 0, 0.2, 1)

--ease-spring

cubic-bezier(0.34, 1.56, 0.64, 1)

When to use each easing

--ease-defaultDefault for most transitions. Starts fast, ends gentle.
--ease-inElements exiting view (fade out, slide off-screen). Starts slow, accelerates away.
--ease-outElements entering view (dropdown open, modal appear). Decelerates to a stop.
--ease-in-outContinuous motion (loading bars, looping animations). Symmetric acceleration.
--ease-springPlayful emphasis (toggle snap, badge bounce). Overshoots then settles. Use sparingly.

Common Transition Patterns

These are the standard transition recipes used across Haven components. Copy the CSS variable usage, not hardcoded values.

Color change

transition: color var(--duration-fast) var(--ease-default),
  background-color var(--duration-fast) var(--ease-default);

Used by: buttons, links, nav items, filter pills

Opacity

transition: opacity var(--duration-fast) var(--ease-default);

Used by: dropdowns, tooltips, skeleton loaders

Shadow lift

transition: box-shadow var(--duration-normal) var(--ease-out);

Used by: card hover, focused inputs

Transform (slide / scale)

transition: transform var(--duration-slow) var(--ease-out);

Used by: sidebar, modals, accordion panels

Reduced Motion

When the user's OS has "Reduce motion" enabled, all Haven transitions should collapse to instant. Add this rule globally or per-component:

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
