Iconography

Haven uses FontAwesome Pro v7.1.0 exclusively. Icons are loaded from a local copy at src/vendor/fontawesome/. Never use CDN links, Material Icons, Heroicons, or emoji as icons.

Usage Rules

LibraryFontAwesome Pro v7.1.0 only
Source/src/vendor/fontawesome/css/all.css (local). Never all.min.css (does not exist) or CDN.
Markup<i class="fa-solid fa-{name}"></i>
Default stylefa-solid for most UI icons. Use lighter weights for decorative/secondary use.
SizingIcons inherit font-size from their parent. Use Tailwind text-size classes to override.
ColorIcons inherit color from their parent. Do not set color on the <i> tag unless overriding.
AccessibilityDecorative icons: no extra attributes needed. Meaningful icons: add aria-label or pair with visible text.
No emojiNever use emoji as icons. Use FA Pro icons for all icon needs.

Sizing

Icons inherit font-size from their parent element. Override with Tailwind text-size utilities when needed. These are the standard sizes used in Haven.

text-xs 12px
text-sm 14px
text-base 16px
text-lg 18px
text-xl 20px
text-2xl 24px
text-3xl 30px
text-4xl 36px

Sizing guidance

text-xsInline with small text, badge icons, dropdown chevrons
text-smInline with body text, button icons, nav items
text-baseDefault. Standalone icons, form field indicators
text-xl to text-2xlCard accent icons, stat card icons, empty state illustrations
text-3xl to text-4xlHero-level icons, empty state focal points. Use sparingly.

Color & Context

Icons inherit color from their parent by default. Apply semantic color classes directly to the <i> element only when the icon needs to differ from its surrounding text.

Color rules

InheritDefault. Let the icon take the parent element's text color. Works in buttons, nav, badges.
Semantic colorApply text-{status}-600 when the icon carries independent meaning (e.g., a check for success).
MutedUse text-gray-400 for decorative or de-emphasized icons.
AI / VioletUse text-violet-600 for AI-generated content indicators.
